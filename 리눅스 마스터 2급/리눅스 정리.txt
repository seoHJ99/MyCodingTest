chown  -R : 하위 파일까지 소유권 변경
           -f : 오류문구 출력 하지 않음
           -h : 심볼릭 파일 변경
           -H : 심볼릭 링크의 참조 파일만 변경

chmod : 파일 권한 변경
         -R : 하위 파일까지 변경
         -v : 진단 메시지 출력
         -c : 기존 권한이 변경되면 메시지 출력
         -f : 메시지 출력하지 않음

touch : 파일 생성 시간 변경
    -t: 지정 시간
    -m : 서버 시간

umask : 파일 기본 권한 변경
    기본값은 0022(022)
    일반 파일은 0644
    디렉토리는 755

/etc/passwd : 사용자 정보 저장

삼바 : 윈도우와 리눅스의 파일 공유 프로그램. 이기종간 파일, cd-rom, 프린터를 공유하기 위해 만들어짐. 
     CIFS 프로토콜 사용
     smb 는 마이크로 소프트 자원 공유 목적 프로토콜. CIFS 로 업그레이드됨.

파일
/etc/fstab : 파일 시스템 정보 저장. 
mkefs -j : ext3
mke2fs -t ext4: ext4
fsck : 파일 시스템 및 일관성 검사 및 복구 가능
      -p : 사용자 개입 업시 자동 복구 
      복구 작업 수행서 임시로 /lost+found 디렉터리에서 작업을 수행
fdisk : 파티션 테이블을 관리
      -l : 현재 디스크 및 파티션
      -n : 신규 파티션 추가
      -t : 파티션 속성 변경
      -d : 파티션 삭제
      -p : 파티션 설정 보기

mkfs : 블록 초기화. 지정된 장치에 새로운 파일 시스템 작성

du : 특정 디렉토리 사용량 확인
df : 리눅스 시스템 전체의 디스크 사용량

chsh -s: 사용중인 쉘 변경 명령어

/etc/passwd : 사용자의 쉘정보 확인
/etc/shell : 설치된 쉘 모두 확인

history :이전 명령어 저장. .bash_history에 저장됨. 
     - w : 저장 위치 변경
history [숫자] : 숫자만큼 최근 명령어 출력
!! : 이전 명령어 재실행
HISTSIZE : 히스토리 스택의 개수를 지정


/etc/profile : 시스템 전체에 적용되는 환경변수와 시작 관련 프로그램 저장
/.bash_profile : 특정 디렉토리를 명령어 검색 디렉토리로 지정하기 위해 환경변수 path 에 등록하고 계속 사용하려고 함
/.bashrc : alias 및 함수들을 설정할때 사용하는 파일
/.bash_logout : 로그아웃할때 자동으로 생성되는 .bak 파일이 저장됨

PS1 : 쉘 프롬프트 모양을 설정할수 있는 환경변수

alias 명령어="별칭" : alias 등록
unalias : alias 해제.


nice : 프로세스명 입력. 실행전. 그냥 이름만 입력하면 10 증가된 상태. 일반 유저는 숫자를 높일수만 있다(숫자가 높을수록 우선순위 낮아짐).
renice : pid 값 입력.  증가가 아니라 해당 숫자 순위로 변경. N 으로 우선순위 바뀜 표시.

PRI 와 NI 차이
PRI : 운영체제가 참고하는 순위. 사용자 조작 불가
NI : 사용자가 조작 가능한 우선순위

init : 시스템 시작시 최초의 프로세스. pid는 1

fork : 기존 프로세스를 복제해 새로운 프로세스 실행. 메모리 할당 필요
exec : 호출하는 프로세스가 새로운 프로세스로 변경되는 방식으로, 메모리 할당 필요없음

cron : 주기적으로 명령어 실행
/etc/crontab : 시스템이 설정한 크론
/var/spool/cron : 사용자가 지정한 크론
crontab -e -u [사용자명] : 특정 사용자(-u)의 크론 내용 수정(-e)
분/시간/일/월/요일 지정

kill -9 : 죽이기

jobs : 작업중인 모든 프로세스(백그라운드도 포함) 표시
fg %[프로세스 pid] : 백그라운드를 포어그라운드로 전환
[명령어, 프로세스] & : 백그라운드에서 명령어나 프로세스 실행. 뒤에 붙인다.

패킷 교환 방식
1. 동적인 대역폭
2. 패킷단위로 분할
3. 배달될때까지 메시지 저장
4. 패킷마다 별도의 전송로
5. 배달 불가하면 알려줌
6. 각 패킷마다 오버헤드 비트


/etc/services : 리눅스에서 사용하는 포트 저장
/etc/sysconfig/network : 네트워크 정보 저장. 시스템의 hostname 과 default GW 를 지정.
/etc/host : 소규모 lan 환경에서 dns 처럼 작동. 1순위
/etc/resolv.conf : 외부 인터넷과 연결될때 dns.

virtualBox : 리눅스 가상화 기술. VT-x와 AMD-V 를 기반으로 전가상화를 지원
XEN : 리눅스 cpu 반가상화 기술. 

리눅스 클러스터
1. HPC - 고계산용. 즉, 베어울프 클러스터. 병렬구조.
2. LVS - 부하분산. 즉, 로드밸런싱
3. HA - 고가용성. 즉, 백업


Ansible : 인프라를 코드 기반으로 자동화하는 도구. 소프트웨어 프로비저닝 관리. 
Tizen : 삼성과 인텔이 주도해서 만든 리눅스 기반 모바일 운영체제.
SSL/TLS 프로토콜 : 응용 계층과 전송 계층 사이의 독립적 계층. OSI 7계층 바깥
QNX : 유닉스계열 실시간 운영체제. 임베디드에서 쓰임. 리눅스와 거리가 멈.
