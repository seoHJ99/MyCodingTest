실습 개요

Kubernetes에서 애플리케이션을 관리하는 것은 어려울 수 있지만 Helm이 도움이 됩니다. Helm은 Linux의 apt 또는 yum과 마찬가지로 Kubernetes에서 패키지 관리자 역할을 합니다. Helm을 사용하여 Kubernetes 애플리케이션을 Helm 차트라는 재사용 가능한 번들로 패키징할 수 있습니다. Helm 차트는 다음과 같이 애플리케이션을 실행하는 데 필요한 관련 Kubernetes 리소스 세트를 설명하는 YAML 파일의 모음입니다.

  • 배포
  • 서비스
  • ConfigMap
  • Secret

차트는 Kubernetes 매니페스트를 동적으로 구성하기 위한 템플릿 작성 구문을 제공합니다. 이를 통해 애플리케이션을 다양한 환경에 맞춰 사용자 지정하고 배포할 수 있습니다.

Helm을 사용하면 다음 작업을 수행할 수 있습니다.

  • 애플리케이션을 공유 가능한 차트로 패키징
  • 차트를 리포지토리에 게시하여 공유
  • 하나의 명령줄 인터페이스(CLI) 명령으로 차트를 검색 및 설치
  • 클러스터에 설치된 차트의 릴리스를 관리
  • 이전 버전으로 롤백

이 실습에서는 Helm을 사용하여 Kubernetes에서 애플리케이션을 패키징, 배포, 관리하는 방법을 배웁니다. 샘플 애플리케이션에 대한 Helm 차트를 생성하여 Amazon Simple Storage Service(Amazon S3) 차트 리포지토리에 푸시한 다음 Kubernetes 클러스터에 애플리케이션을 설치합니다.

목표

이 실습을 마치면 다음을 수행할 수 있습니다.

  • Helm 차트의 기본 구조를 정의합니다.
  • Helm 리포지토리로 사용할 Amazon S3 버킷을 설정합니다.
  • 차트를 생성하고 S3 리포지토리에 업로드합니다.
  • S3 리포지토리에서 Helm 차트를 패키징 및 설치합니다.

실습 환경

다음 이미지에는 설계된 솔루션의 최종 아키텍처가 나와 있습니다.

Helm은 apt 또는 yum과 같은 친숙한 Linux 리포지토리와 마찬가지로 Kubernetes의 패키지 관리자로, Kubernetes 클러스터에 애플리케이션을 설치하고 관리하는 데 도움이 됩니다. Helm은 차트라는 패키징 형식을 사용합니다. Helm 차트는 관련 Kubernetes 리소스 세트를 설명하는 파일의 모음입니다.

이 실습에서는 다음 서비스를 사용합니다.

  • Amazon Elastic Kubernetes Service(Amazon EKS)
  • Amazon Elastic Compute Cloud(Amazon EC2)
  • Amazon S3

이 실습에서 사용하지 않는 AWS 서비스는 실습 환경에서 꺼져 있습니다. 다른 서비스에 액세스하거나 이 실습 가이드에서 제공하는 것 이외의 작업을 수행하는 경우 오류가 발생할 수 있습니다.

과제 1: Helm을 설치하고 Amazon S3 버킷을 Helm 리포지토리로 생성

이 과제에서는 Helm을 설치하고 Amazon S3 버킷을 Helm 리포지토리로 구성합니다.

Helm을 사용하여 Kubernetes 애플리케이션을 Helm 차트라는 재사용 가능한 번들로 패키징할 수 있습니다. 차트는 Pod, 서비스, 수신 등 앱이 실행하는 데 필요한 모든 리소스를 정의합니다.

환경 간 공유 및 재사용을 위해 차트를 Helm 차트 리포지토리에 저장합니다. Helm-s3 플러그 인을 사용하면 S3 버킷을 차트 리포지토리로 사용할 수 있습니다. Amazon S3를 Helm 리포지토리로 사용하면 다음과 같은 여러 이점이 있습니다.

  • Amazon S3는 보안, 내구성 및 확장성이 뛰어난 객체 스토리지를 제공합니다. 이는 많은 Helm 차트 패키지를 저장하는 데 이상적입니다.
  • Amazon S3는 차트를 구성하고, 권한을 관리하고, 백업을 생성하는 데 유용합니다.
  • S3 버킷에 대한 액세스 권한이 있는 모든 환경에서 차트에 액세스할 수 있습니다. 자체 파일 서버를 실행할 필요가 없습니다.
  • S3의 버전 관리 기능은 경시적으로 차트 변경 사항을 추적하는 데 도움이 됩니다.

  1. AWS Management Console에서 AWS 검색 창 을 사용하여

    Copied
    를 검색한 다음 결과 목록에서 해당 서비스를 선택합니다.

  2. EC2 Dashboard의 Resources 섹션에서 Instances (running) 를 선택합니다.

  3. Bastion Host 인스턴스를 선택한 다음 Connect를 선택합니다.

  4. Connect to instance 페이지에서 Session Manager 탭을 선택한 다음, Connect를 선택합니다.

배스천 호스트 인스턴스가 포함된 터미널 세션이 새 탭에서 열립니다.

과제 2: 제품 카탈로그 차트를 패키징하고 Amazon S3로 푸시

이 과제에서는 샘플 애플리케이션을 Helm 차트로 패키징하고 앞서 생성한 Amazon S3 리포지토리에 푸시합니다.

먼저 GitHub에서 애플리케이션 코드를 복제합니다. 이 코드에는 애플리케이션을 배포하는 데 필요한 Kubernetes 리소스 매니페스트가 포함되어 있습니다. 다음으로 Helm CLI를 사용하여 이러한 리소스를 단일 Helm 차트 아카이브로 패키징합니다. 그러면 애플리케이션을 재사용 가능한 형식으로 실행하는 데 필요한 모든 것이 번들로 제공됩니다. helm package 명령은 차트 파일을 버전이 지정된 .tgz 파일로 압축합니다. 차트 버전 번호는 차트에 포함된 Chart.yaml 파일에서 가져옵니다.

차트 패키징은 다음 작업을 수행할 수 있기 때문에 유용합니다.

  • S3 버킷과 같은 Helm 차트 리포지토리에서 차트를 공유
  • 단일 Helm instal 명령으로 패키징된 차트를 설치
  • 환경 간에 업데이트 및 롤백을 관리

차트를 패키징한 후 Helm-s3 플러그 인을 사용하여 Helm 차트 리포지토리로 구성된 S3 버킷에 푸시합니다.

과제 3: Amazon S3 리포지토리에서 Helm 차트를 검색 및 설치

이 과제에서는 이전 단계에서 Amazon S3에 푸시한 Helm 차트를 배포하여 샘플 애플리케이션을 설치합니다.

먼저 Helm 명령줄 인터페이스(CLI)를 사용하여 S3 Helm 리포지토리에서 특정 차트 버전을 검색합니다. 그런 다음 helm install 명령을 사용하여 EKS 클러스터에 차트를 설치합니다. 그러면 S3에서 차트 패키지를 가져와 애플리케이션을 배포합니다.

이 과제를 완료하면 EKS 클러스터의 Pod에서 실행되는 Frontend, Productcatalog 및 Productdetail 서비스가 포함된 샘플 애플리케이션을 얻게 됩니다.

  1. 배스천 호스트에 연결된 Session Manager 세션으로 돌아갑니다.

  2. 명령: S3 버킷에 저장된 Helm 차트의 버전 수를 확인하기 위해 다음 명령을 입력합니다.

  1. 명령: 리소스를 배포하지 않고 차트 설치를 테스트하기 위해 다음 helm install 명령을 –dry-run 플래그와 함께 입력합니다.
Copied

참고: 여기서 명령이 –dry-run 및 –debug 플래그를 사용하는 것을 볼 수 있습니다. 이러한 플래그를 사용하면 Kubernetes API 서버에 연결하여 차트를 검증 및 확인할 수 있습니다. 다음은 이 명령 및 그 안에 포함된 인수에 대한 설명입니다.

  • productcatalog: 이 설치에 부여하는 릴리스 이름입니다.
  • s3://$S3_BUCKET_NAME/productcatalog_workshop-1.0.0.tgz: S3 버킷 리포지토리에서 차트 아카이브의 위치를 지정합니다.
  • –version 1.0.0: 설치할 차트 버전을 1.0.0으로 고정합니다.
  • –dry-run: Helm에게 차트를 검증하고 실제로 배포하지 않고 리소스 매니페스트를 생성하도록 지시합니다.
  • –debug: 설치 프로세스에 대한 자세한 디버깅 정보를 출력합니다.

sed -i "s/replicaCount:.*/replicaCount: 3/g" values.yaml
