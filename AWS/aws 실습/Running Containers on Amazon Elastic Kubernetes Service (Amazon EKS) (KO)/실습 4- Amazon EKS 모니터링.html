실습 개요

이 실습에서는 Amazon Elastic Kubernetes Service(Amazon EKS) 클러스터에서 실행되는 애플리케이션용 중앙 집중형 로깅 솔루션을 생성하는 방법을 배웁니다. Fluent Bit 로깅 에이전트를 DaemonSet 애플리케이션으로 Amazon EKS 클러스터 노드에 배포합니다. Fluent Bit는 로그 파일, 컨테이너, 시스템 지표 등 다양한 소스에서 애플리케이션 데이터를 수집한 다음, 이 데이터를 Amazon OpenSearch Service 도메인 내의 대상으로 라우팅합니다. 이 데이터는 Fluent Bit를 통해 Amazon Data Firehose로 스트리밍되고, 여기서 로그 데이터가 Amazon OpenSearch 도메인으로 전달됩니다. 그런 다음 OpenSearch 대시보드에 로그인하고 Fluent Bit에서 OpenSearch 클러스터로 스트리밍된 데이터에 대한 검색을 수행합니다.

목표

이 실습을 마치면 다음을 수행할 수 있습니다.

  • Fluent Bit DaemonSet를 사용하여 Amazon EKS 클러스터에서 로그를 라우팅합니다.
  • Amazon Data Firehose를 통해 Amazon Simple Storage Service(Amazon S3) 버킷으로 데이터를 구성하고 스트리밍합니다.
  • Amazon OpenSearch Service를 사용하여 Amazon S3 버킷의 데이터를 구성하고 분석합니다.
  • Prometheus를 사용하여 컨테이너식 애플리케이션 지표를 수집 및 요약하고 Grafana를 사용하여 확인합니다.

이 실습에서 사용되는 서비스들

Amazon Elastic Kubernetes Service(Amazon EKS)

Amazon EKS는 Amazon Web Services(AWS)에서 자체 Kubernetes 제어 영역을 설치, 운영, 유지 관리할 필요가 없는 관리형 서비스입니다.

Amazon OpenSearch Service

Amazon OpenSearch Service는 애플리케이션 모니터링, 로그 분석, 관찰 기능, 웹 사이트 검색과 같은 사용 사례를 위한 비즈니스 및 운영 데이터의 실시간 검색, 모니터링 및 분석을 안전하게 잠금 해제합니다. AWS가 가동 중단 없이 소프트웨어 설치, 업그레이드, 패치 적용, 크기 조정(최대 3페타바이트[PB]) 및 리전 간 복제를 관리합니다. 또한 OpenSearch Service에는 로그 및 추적 데이터를 시각화하는 데 도움이 되는 대시보드 시각화 도구인 OpenSearch 대시보드와 이상 탐지 및 검색 관련성 순위를 위한 기계 학습 기반 결과도 번들로 제공됩니다.

Amazon Data Firehose

Amazon Data Firehose는 스트리밍 데이터를 신뢰할 수 있게 캡처하고 변환한 후 데이터 레이크, 데이터 스토어 및 분석 서비스로 전달하는 추출, 변환, 로드(ETL) 서비스로, Amazon S3, Amazon Redshift, Amazon OpenSearch Service, Splunk와 같은 대상 및 사용자 지정 HTTP 엔드포인트 또는 지원되는 서드 파티 서비스 공급업체(Datadog, Dynatrace, LogicMonitor, MongoDB, New Relic, Sumo Logic 등)에서 소유한 HTTP 엔드포인트에 실시간 스트리밍 데이터를 제공하기 위한 완전관리형 서비스입니다. Kinesis Data Firehose는 Kinesis Data Streams, Kinesis Video Streams, Amazon Kinesis Data Analytics와 함께 Kinesis 스트리밍 데이터 플랫폼의 일부입니다. Kinesis Data Firehose를 사용하면 애플리케이션을 작성하거나 리소스를 관리할 필요가 없습니다. 데이터를 Kinesis Data Firehose로 전송하도록 데이터 생산자를 구성하면 Firehose에서 자동으로 데이터를 사용자가 지정한 대상으로 전송합니다. 데이터를 전송하기 전에 변환하도록 Kinesis Data Firehose를 구성할 수도 있습니다

과제 2: Prometheus 배포 및 구성

이 과제에서는 Prometheus를 배포하고 구성합니다. Prometheus는 원래 SoundCloud에서 개발한 시계열 기반 오픈 소스 시스템 모니터링 도구입니다. Prometheus는 2016년에 Kubernetes에 이어 두 번째로 Cloud Native Computing Foundation에 호스팅된 프로젝트로 합류했습니다.

Prometheus는 HTTP를 통한 풀 모델을 사용하여 지표를 수집합니다. Kubernetes에서 Prometheus는 Kubernetes API를 사용하여 자동으로 대상을 검색할 수 있습니다. 대상은 Pod, 대몬 세트, 노드 등이 될 수 있습니다. Kubernetes에서 일반적인 Prometheus 설치에는 다음 구성 요소가 포함됩니다.

  • Prometheus 서버
  • Node Exporter
  • Push Gateway
  • Alert Manager
  • kube-state-metrics

Kubernetes에서 Prometheus 서버는 지표를 스크래핑하여 로컬 시계열 데이터베이스에 저장하는 역할을 하는 Pod로 실행됩니다. 컨테이너 재시작 후 이 데이터베이스에 저장된 지표가 손실되지 않도록 하려면 먼저 Amazon Elastic Block Store(Amazon EBS) Container Storage Interface(CSI) 드라이버 를 설치하고 영구 볼륨을 탑재합니다.
