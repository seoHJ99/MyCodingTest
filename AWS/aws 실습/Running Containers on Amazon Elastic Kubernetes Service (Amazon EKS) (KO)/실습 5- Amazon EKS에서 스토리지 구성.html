실습 개요

스테이트풀 애플리케이션을 관리하고 데이터를 유지하는 것은 Kubernetes에서 프로덕션 워크로드를 실행하는 데 매우 중요합니다. 이 실습에서는 Amazon Elastic File System(Amazon EFS)을 사용하여 Amazon Elastic Kubernetes Service(Amazon EKS)에서 스테이트풀 애플리케이션을 위한 영구 스토리지를 구성하는 방법을 배웁니다.

Amazon EFS는 Amazon EKS에서 실행되는 Kubernetes 애플리케이션이 사용할 수 있는 확장 가능하고 탄력적인 공유 파일 시스템을 제공합니다. Amazon EFS Container Storage Interface(CSI) 드라이버를 사용하면 Kubernetes가 Amazon EFS 파일 시스템을 동적으로 프로비저닝하고 Pod에 볼륨으로 탑재할 수 있습니다.

이 실습에서는 다음을 수행합니다.

  • EKS 클러스터에 Amazon EFS CSI 드라이버를 설치합니다.
  • EFS 파일 시스템 및 탑재 대상을 생성합니다.
  • StorageClass, PersistentVolume 및 PersistentVolumeClaim을 구성합니다.
  • 샘플 애플리케이션을 배포하고 EFS 볼륨을 탑재합니다.
  • 공유 EFS 파일 시스템에 대한 데이터 읽기 및 쓰기를 테스트합니다.

Amazon EFS를 사용하여 영구 스토리지를 구성하면 Amazon EKS에서 데이터베이스, 캐시, 파일 공유 애플리케이션과 같은 스테이트풀 애플리케이션을 안정적으로 실행할 수 있습니다. Amazon EFS CSI 드라이버를 사용하면 여러 Pod에서 EFS 스토리지를 동적으로 프로비저닝, 탑재, 공유할 수 있습니다.

목표

이 실습을 마치면 다음을 수행할 수 있습니다.

  • 분산 워크로드를 위한 Amazon EFS를 설정하고 구성합니다.
  • 영구 볼륨 클레임, 영구 볼륨 및 스토리지 클래스를 설정합니다.
  • Amazon EFS CSI 드라이버의 기능을 이해합니다.
  • 지속성 파일을 보고 업데이트합니다.

먼저 스토리지 볼륨을 제공하는 새 EFS 파일 시스템을 생성합니다. 그런 다음 VPC 내의 컴퓨팅 인스턴스에서 EFS 파일 시스템에 연결하기 위한 액세스 포인트를 제공하는 탑재 대상을 생성합니다. 이 과제를 완료하면 EFS 파일 시스템이 설정되고 EKS 클러스터에서 스테이트풀 애플리케이션을 위한 영구 스토리지로 사용할 수 있게 됩니다.

과제 2: 지속성 구성 요소 생성 및 Amazon EFS CSI 드라이버 설치

이 과제에서는 Amazon EFS 및 Amazon EFS CSI 드라이버를 사용하여 Amazon EKS에서 실행되는 애플리케이션을 위한 영구 스토리지를 구성합니다. 이 드라이버는 Amazon EFS 파일 시스템의 수명 주기를 관리하는 컨테이너 오케스트레이터에 대한 CSI 사양을 구현합니다.

Amazon EFS CSI 드라이버는 EFS 파일 시스템의 동적 및 정적 프로비저닝을 모두 지원합니다. 동적 프로비저닝의 경우 먼저 EFS 파일 시스템을 수동으로 생성해야 합니다. Kubernetes의 PersistentVolume(PV) 하위 시스템은 스토리지 프로비저닝과 스토리지 사용을 분리하는 추상화를 제공합니다. PersistentVolume은 클러스터의 스토리지 리소스입니다. PersistentVolumeClaims은 사용자의 스토리지 요청입니다. Product Catalog 애플리케이션에 영구 스토리지를 제공하려면 Amazon EFS CSI 드라이버를 사용해야 합니다. EFS 파일 시스템을 수동으로 생성하고 StorageClass에서 해당 시스템을 참조해야 합니다. 이 StorageClass에서 스토리지를 프로비저닝하려면 PersistentVolume 및 PersistentVolumeClaim(PVC)을 구성해야 합니다. 애플리케이션은 제품 데이터를 유지하기 위한 PVC를 탑재해야 합니다.

Helm을 사용하여 Amazon EFS CSI 드라이버를 설치합니다 그런 다음 StorageClass 및 PVC를 생성하여 EFS 파일 시스템을 프로비저닝합니다. Product Catalog 애플리케이션은 제품 데이터를 유지하는 데 이 PVC를 사용해야 합니다.

  1. 배스천 호스트에 연결된 Session Manager 세션으로 돌아갑니다.

  2. 명령: Helm을 사용하여 Amazon EFS CSI 드라이버를 설치하기 위해 다음 명령을 입력합니다.

참고: prodcatalog 서비스의 복제본은 현재 1개입니다. Amazon EFS에는 ReadWriteMany 액세스 모드를 사용하여 동일한 영구 볼륨을 여러 Pod에 동시에 탑재할 수 있는 기능이 있습니다. Amazon EFS에 탑재된 데이터가 Pod 간에 지속되는지 테스트하려면 prodcatalog 서비스를 2개의 복제본으로 확장하고 두 번째 Pod가 공유 영구 볼륨에서 동일한 데이터를 성공적으로 읽을 수 있는지 확인합니다.

  1. 명령: prodcatalog 서비스를 2개의 복제본으로 확장하기 위해 다음 명령을 입력합니다.
Copied 
kubectl scale --replicas=2 deployment/prodcatalog -n workshop

예상 출력:

Copied 
************************
**** EXAMPLE OUTPUT ****
************************

deployment.apps/prodcatalog scaled
  1. 명령: 새 Pod가 생성되는지 확인하기 위해 다음 명령을 입력합니다.
Copied 
kubectl get pods -n workshop -l app=prodcatalog

예상 출력:

Copied 
************************
**** EXAMPLE OUTPUT ****
************************

NAME                           READY   STATUS              RESTARTS   AGE
prodcatalog-xxx-htkxx   1/1     Running             0          16m
prodcatalog-xxx-nmzmd   0/1     ContainerCreating   0          4s

주의: 몇 분 정도 기다렸다가 새 Pod가 성공적으로 생성되고 Running 상태인지 확인한 후 다음 단계로 진행하십시오.

이제 products.txt 파일을 검토할 수 있도록 새 Pod로 exec합니다.

  1. 명령: Pod로 exec하기 위해 다음 명령을 입력합니다.
Copied 
NEW_POD=$(kubectl get pods -n workshop -l app=prodcatalog --sort-by=.metadata.creationTimestamp -o jsonpath='{.items[-1:].metadata.name}')
kubectl -n workshop exec -it $NEW_POD -c prodcatalog -- /bin/bash
  1. 명령: 새 복제본 Pod에 ID가 1인 EKS-Course가 존재하는지 확인하기 위해 다음 명령을 입력합니다.
Copied 
cat /products/products.txt

예상 출력:

Copied 
************************
**** EXAMPLE OUTPUT ****
************************

1 EKS-Course

참고: 이 출력은 EFS 스토리지를 사용하는 여러 Pod 간의 데이터 지속성을 보여줍니다. 즉, 새로 생성된 Pod가 기존 Pod에서 기록한 동일한 데이터에 액세스할 수 있습니다.

특히, 다음을 확인할 수 있습니다.

  • 원래 Pod가 제품 EKS-Course(ID 1)를 Amazon EFS 볼륨에 기록했습니다.
  • 이 데이터가 Amazon EFS에서 유지되었습니다.
  • 배포를 확장하여 새 Pod가 생성되었습니다.
  • 새 Pod가 동일한 Amazon EFS 볼륨을 탑재합니다.
  • 새 Pod에서 cat /products/products.txt를 실행하면 이 파일이 첫 번째 Pod에서 기록한 데이터에 액세스할 수 있습니다.

이를 통해 Amazon EFS는 Pod가 종료되고 다시 생성되는 경우에도 Pod가 동일한 데이터에 액세스할 수 있도록 하는 영구적인 공유 스토리지 볼륨을 제공한다는 것을 알 수 있습니다.

과제 완료: Amazon EFS의 영구 볼륨 동작을 테스트했습니다.

마무리
