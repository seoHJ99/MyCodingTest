목표

이 실습을 마치면 다음을 수행할 수 있습니다.

  • Grafana 대시보드를 사용하여 Pod 지표 문제를 해결합니다.
  • 일반적인 Pod 오류를 식별하고 해결합니다.
  • Amazon Simple Storage Service(Amazon S3) 내에 저장된 Helm 차트를 마이그레이션, 업데이트, 배포합니다.
  • Amazon EFS Container Storage Interface(CSI) 드라이버를 설치하고 확장 가능한 방식으로 Amazon EFS 볼륨을 Pod에 탑재합니다.

이 실습에서 사용되는 서비스들

Amazon Elastic Kubernetes Service(Amazon EKS)

Amazon EKS는 Amazon Web Services(AWS)에서 자체 Kubernetes 제어 영역을 설치, 운영, 유지 관리할 필요가 없는 관리형 서비스입니다.

Amazon Simple Storage Service(S3)

Amazon S3는 데이터 용량에 관계없이 확장 가능하고 내구성이 뛰어난 스토리지를 제공하는 객체 스토리지 서비스입니다. S3에서는 웹 서비스 API를 사용하여 파일 및 객체를 저장하고 검색할 수 있습니다.

Amazon Elastic Container Registry(Amazon ECR)

Amazon ECR은 Docker 컨테이너 이미지를 저장, 관리, 배포하기 위한 완전관리형 Docker 컨테이너 레지스트리입니다. Amazon ECR은 Amazon ECS 및 AWS Fargate와 통합되어 컨테이너 기반 애플리케이션의 이미지를 가져올 수 있습니다.

Amazon Elastic File System(Amazon EFS)

Amazon EFS는 클라우드의 Linux 기반 워크로드를 위한 확장 가능하고 탄력적인 파일 시스템을 제공합니다. EFS를 사용하면 여러 가용 영역에 걸쳐 EC2 인스턴스 또는 컨테이너에 파일 시스템을 탑재하여 액세스 공유 및 고가용성을 구현할 수 있습니다.

Amazon Kinesis Data Firehose

Amazon Kinesis Data Firehose는 실시간 스트리밍 데이터를 S3, Amazon Redshift, Amazon OpenSearch Service, Splunk와 같은 대상에 전송하는 완전관리형 서비스입니다. 스트리밍 데이터를 안정적으로 캡처, 변환, 로드할 수 있으며 수십만 개의 소스에서 데이터를 수집하는 데 필요한 처리량에 맞춰 자동으로 확장됩니다. Kinesis Data Firehose는 실시간 데이터 스트리밍 및 분석을 위한 인프라 관리 부담을 덜어줍니다.
