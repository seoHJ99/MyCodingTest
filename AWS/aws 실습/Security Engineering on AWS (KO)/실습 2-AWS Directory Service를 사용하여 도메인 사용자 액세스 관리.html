실습 개요

이 실습에서는 Windows 기반 호스트를 사용하여 AWS Directory Service Simple Active Directory(Simple AD) 디렉터리에 두 명의 사용자를 생성합니다. 그런 후 각 사용자를 특정 Identity and Access Management(IAM) 역할에 할당합니다. 다음으로 사용자가 생성한 각 디렉터리 사용자의 권한으로 AWS 관리 콘솔에 로그인하여 IAM 역할에 의해 할당된 유효 권한을 검토합니다. 마지막으로 악의적인 작업을 수행한 디렉터리 사용자를 격리하여 이러한 사용자가 로그인하여 회사 자산에 더 많은 손해를 입히지 않도록 합니다.

목표

본 실습을 마치면 다음을 할 수 있습니다.

  • 사용자 및 그룹을 AWS Directory Service Simple AD 디렉터리에 추가합니다.
  • 디렉터리 사용자를 IAM 역할에 연결하여 AWS 관리 콘솔 액세스 권한을 정의합니다.
  • 디렉터리 사용자 권한으로 AWS 관리 콘솔에 로그인합니다.
  • 악의적인 사용자의 액세스 권한을 제거합니다.

필수 기술 지식

이 실습을 정상적으로 완료하려면 다음 AWS Cloud Practitioner Essentials 과정에 정의된 것처럼 AWS 서비스에 대한 기본적인 지식을 보유하고 있어야 합니다. AWS 관리 콘솔을 능숙하게 탐색하고 기본 사용자 디렉터리 개념을 숙지해야 합니다.

시나리오

여러분은 AnyCompany의 보안 엔지니어로서 IAM 사용자 목록을 유지할 필요 없이 회사 직원들이 회사 자격 증명으로 AWS 관리 콘솔에 로그인할 수 있게 하려고 합니다. 특정 직무 역할만 AWS 관리 콘솔과 상호 작용하도록 허용되며 해당 권한은 일별 태스크를 수행하는 데 필요한 최소 권한으로 제한해야 합니다. 시작하려면 두 가지 유형의 역할을 생성할 수 있습니다. 하나는 인프라 관리 팀이 Amazon Elastic Compute Cloud(Amazon EC2) 인스턴스를 볼 수 있도록 허용하는 역할이고 다른 하나는 스토리지 관리자가 Amazon Simple Storage Service(Amazon S3) 버킷을 관리할 수 있도록 허용하는 역할입니다. 사용자가 회사 정책을 위반한 것이 확인될 경우 해당 액세스 권한을 빠르게 제거하려고 할 수 있습니다.

태스크 2: 디렉터리 사용자를 IAM 역할에 연결

이 태스크에서는 사용자에 대한 AWS 관리 콘솔 액세스를 허용하도록 AWS Directory Service를 구성합니다. 애플리케이션 액세스 URL을 생성하고 디렉터리 그룹을 AWS 리소스에 대한 액세스 권한을 정의하는 AWS IAM 역할에 연결합니다.

  1. AWS 관리 콘솔을 사용하여 웹 브라우저 탭으로 돌아갑니다.

  2. AWS 관리 콘솔 상단의 검색 창에서

    Copied
    를 검색하여 선택합니다.

  3. Active Directory의 Directories 페이지에서 lab.local 디렉터리에 대한 Directory ID 링크를 선택하여 해당 세부 정보를 확인합니다.

  4. Directory details 페이지에서 Application management 탭을 선택합니다.

  5. Application access URL 섹션 상단 오른쪽 구석에 있는 Create를 선택합니다.

  6. Create application access URL 팝업 창에서 다음을 수행합니다.

  • Access URL name에 이 지침의 왼쪽에 나열된 AppAccessUrl 값을 복사하여 붙여넣습니다.

  1. Create를 선택합니다.

  2. AWS Management Console 섹션에서 Actions , Enable을 순서대로 선택합니다.

  3. Delegate console access 섹션에서 Ec2ReadRole 링크를 선택합니다.

실습 환경 배포 프로세스 동안 Ec2ReadRole IAM 역할이 생성되었습니다. 이 역할에는 Amazon Elastic Compute Cloud(Amazon EC2) 서비스의 모든 측면에 대해 읽기 전용 액세스 권한을 부여하는 AWS 관리형 AmazonEC2ReadOnlyAccess 정책이 포함되며 다음 권한을 포함합니다.

Copied 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "ec2:Describe*",
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": "elasticloadbalancing:Describe*",
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": [
          "cloudwatch:ListMetrics",
          "cloudwatch:GetMetricStatistics",
          "cloudwatch:Describe*"
      ],
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": "autoscaling:Describe*",
      "Resource": "*"
    }
  ]
}

실습 환경 배포 프로세스 동안 S3ReadRole IAM 역할이 생성되었습니다. 이 역할에는 Amazon Simple Storage Service(Amazon S3) 서비스의 모든 측면에 대해 읽기 전용 액세스 권한을 부여하는 AWS 관리형 AmazonS3ReadOnlyAccess 정책이 포함되며 다음 권한을 포함합니다.
