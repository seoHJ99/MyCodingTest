실습 개요

이 실습에서는 AWS Key Management Service(AWS KMS) 키를 사용하여 AWS Secrets Manager 보안 암호를 암호화하는 방법을 알아봅니다. 그런 후 이 보안 암호를 사용하여 Amazon Relational Database Service(Amazon RDS) 데이터베이스에 연결합니다.

  • Secrets Manager 보안 암호의 순환을 구성합니다.
  • 암호화된 보안 암호를 사용하여 Amazon RDS 데이터베이스에 연결합니다.

태스크 1: 보안 암호를 사용하여 Amazon RDS 데이터베이스에 액세스할 수 있는지 확인

AWS Secrets Manager 보안 암호를 생성하여 자격 증명을 저장하기 전에 Amazon RDS 데이터베이스에 수동으로 연결할 수 있는지 확인합니다.

  1. 목록에서 이 지침의 왼쪽에 있는 CommandHostSessionUrl 값을 복사한 다음, 새 웹 브라우저 탭에 붙여넣습니다.

콘솔은 웹 브라우저 창 내부의 인스턴스에 연결됩니다. 인스턴스에 연결하면 일련의 명령이 자동으로 실행되어 사용자의 홈 디렉터리로 변경하고 다음과 같이 작업 디렉터리의 경로를 표시합니다.

스크립트 또는 애플리케이션에 하드 코딩 자격 증명을 나타낼 수 있는 표준 사용자 이름 및 암호 방법을 사용하여 데이터베이스에 연결합니다.

  1. 다음 명령을 실행하여 사용자 이름 및 암호를 통해 dbadmin 권한으로 데이터베이스에 연결합니다.
  • DB_ENDPOINT 자리 표시자 값을 이러한 지침의 왼쪽에 나열된 DbEndpoint 값으로 변경합니다.
  • 암호를 입력하라는 메시지가 표시되면 이 지침의 왼쪽에 나열된 DbPassword 값을 복사하여 붙여넣습니다.
Copied 
mysql -u dbadmin -p -h DB_ENDPOINT

태스크 4: 암호화된 보안 암호를 사용하여 Amazon RDS 인스턴스에 연결

이 태스크에서는 셸 스크립트를 생성하고 사용하여 프로그래밍 방식으로 Amazon RDS 데이터베이스에 연결합니다.

  1. 터미널 세션이 EC2 인스턴스에 연결된 상태에서 웹 브라우저 탭으로 돌아갑니다.

  2. 다음 명령을 실행하여 rds-dbadmin-secret 보안 암호 값을 검색합니다.

Copied 
aws secretsmanager get-secret-value --secret-id dbadmin_secret

예상 출력: 출력에는 다음과 같이 보안 암호의 세부 정보가 JSON 형식으로 표시됩니다.

Copied 
{
    "ARN": "arn:aws:secretsmanager:us-west-2:111122223333:secret:dbadmin_secret-a7MiSg",
    "Name": "dbadmin_secret",
    "VersionId": "9e7ff71b-b47b-40fc-815f-34a930bd07e1",
    "SecretString": "{\"username\":\"dbadmin\",\"password\":\"xxxxxxxxxxxxxxxxxxxx\",\"engine\":\"mysql\",\"host\":\"lab-rds-cluster.cluster-xxxxxxxxxxxx.us-west-2.rds.amazonaws.com\",\"port\":3306,\"
dbClusterIdentifier\":\"lab-rds-cluster\"}",
    "VersionStages": [
        "AWSCURRENT"
    ],
    "CreatedDate": "2022-10-18T20:29:59.630000+00:00"
}

보안: 할당된 IAM 인스턴스 프로파일에 보안 암호 및 연결된 KMS 키에 액세스하기 위한 적절한 IAM 정책이 있으므로 이 EC2 인스턴스에서 aws secretsmanager get-secret-value 명령을 실행할 수 있습니다. 프로덕션 환경에서는 항상 보안 암호와 상호 작용해야 할 수 있는 모든 사용자 또는 AWS 서비스에 대해 최소 권한을 적용해야 합니다.

특히 관심 있는 보안 암호 부분은 SecretString 속성의 JSON 문자열 일부로 저장되는 암호입니다. 해당 암호를 복사하고 데이터베이스에 수동으로 연결하는 데 사용할 수 있지만 암호를 프로그래밍 방식으로 검색하고 애플리케이션에 전달하는 여러 방법이 있습니다.

AWS Secrets Manager에서 보안 암호를 검색하는 데 관한 자세한 내용은 이 실습 끝에 있는 추가 리소스 섹션에서 AWS Secrets Manager에서 보안 암호 검색을 참조하십시오.

이 실습에서는 jq 유틸리티를 사용하여 보안 암호 값에서 특정 정보를 구문 분석한 다음, 구문 분석한 정보를 데이터베이스에 연결하는 셸 스크립트를 생성합니다.

  1. 다음 명령을 실행하여 지정된 보안 암호에 대한 보안 암호 값을 검색한 다음, 해당 값을 사용하여 Amazon RDS 데이터베이스에 연결하는 셸 스크립트를 생성합니다.
Copied 
cat<<'EOF'>> get-secret.sh
getsecretvalue() {
  aws secretsmanager get-secret-value --secret-id $1 | \
    jq .SecretString | \
    jq fromjson
}

secret=`getsecretvalue $1`

user=$(echo $secret | jq -r .username)
password=$(echo $secret | jq -r .password)
endpoint=$(echo $secret | jq -r .host)
port=$(echo $secret | jq -r .port)

mysql \
-p$password \
-u $user \
-P $port \
-h $endpoint
EOF

이 스크립트는 다음 작업을 수행합니다.

  • 다음을 수행하는 getsecretvalue() 라는 셸 함수를 정의합니다.
    • AWS CLI secretsmanager get-secret-value 명령을 사용하여 스크립트 입력으로 식별되는 AWS Secrets Manager 보안 암호의 값을 검색합니다.

태스크 5: 보안 암호를 순환하고 Amazon RDS 인스턴스에 다시 연결

이 태스크에서는 보안 암호 순환을 켜서 설정된 간격으로 dbadmin 계정에 대한 암호를 자동으로 변경합니다.

태스크 5.1: 보안 암호 순환 구성

먼저, 데이터베이스 보안 암호에 대한 보안 암호 순환을 구성합니다.

  1. AWS Secrets Manager Secrets 페이지가 있는 웹 브라우저 탭으로 돌아갑니다.

  2. 세부 정보를 보려면 dbadmin_secret 링크를 선택합니다. dbadmin_secret 링크가 보이지 않는다면, 새로고침 아이콘을 클릭합니다.

  3. dbadmin_secret 페이지의 Rotation 탭을 선택 후 Rotation configuration 섹션에서 Edit rotation을 선택합니다.

  4. Edit rotation 팝업 창의 Configure automatic rotation 섹션에서 Automatic rotation을 On 위치로 전환합니다.

  5. Rotation schedule 섹션에서 다음을 수행합니다.

  • Schedule expression builder를 아직 선택하지 않은 경우 선택합니다.
  • Time unit에서 Days를 선택합니다.
  • Days에 대해
    Copied
    을 입력합니다.
  • Rotate immediately when the secret is stored. The next rotation will begin on your schedule.를 선택 취소합니다.

참고: 성공적으로 순환하려면 보안 암호 순환 Lambda 함수는 먼저 보안 암호를 암호화하는 데 사용한 AWS KMS 키에 대한 액세스 권한이 있어야 합니다. 보안 암호를 즉시 순환하려고 할 경우 실패합니다.

  1. Rotation function 섹션에서 다음을 수행합니다.
  • Create a rotation function을 아직 선택하지 않은 경우 선택합니다.
  • Lambda rotation function의 SecretsManager 오른쪽에 있는 텍스트 상자에
    Copied
    을 입력합니다.
  1. Save를 선택합니다.

예상 출력: 페이지 상단에 다음과 같은 내용의 파란색 배너가 표시됩니다(표시하기 위해 위로 스크롤해야 할 수 있음). Creating AWS CloudFormation resources for rotation. 이 프로세스가 완료되면 배너가 녹색으로 바뀌고 메시지는 다음과 같이 변경됩니다.Your secret has been stored and secret rotation is enabled.

새로 고침: Lambda 함수 및 보안 암호를 순환하는 데 필요한 관련 리소스가 생성을 완료하는 데 약 3~5분이 걸릴 수 있습니다. 웹 브라우저의 새로 고침 버튼을 사용하여 페이지의 세부 정보를 새로 고쳐야 할 수 있습니다. 순환 상태가 계속 Disabled로 표시되면 2분 동안 기다렸다가 페이지를 다시 새로 고칩니다.

태스크 5.2: 보안 암호 순환 함수가 AWS KMS 키에 액세스하기 위한 권한 부여

지금까지 보안 암호 순환을 수행하기 위해 Lambda 함수를 생성했으므로 보안 암호를 암호화하는 데 사용한 AWS KMS 키에 대한 액세스 권한을 부여해야 합니다. Lambda가 키에 액세스하도록 허용하지 않을 경우 보안 암호를 성공적으로 순환할 수 없습니다.

  1. AWS 관리 콘솔 상단의 검색 창에서

    Copied
    를 검색하여 선택합니다.

  2. Customer managed keys 페이지에서 rds_secret_key 링크를 선택하여 해당 세부 정보를 확인합니다.

  3. Key policy 탭의 Key users 섹션에서 Add를 선택합니다.

  4. Add key users 팝업 창에서 보안 암호 순환을 켤 때 Secrets Manager가 생성한 IAM 역할에 해당하는

    Copied
    로 시작하는 역할을 검색한 후 선택합니다.

  5. Add를 선택합니다.

태스크 5.3: 보안 암호 순환 수행

지금까지 보안 암호 순환 함수에 할당된 IAM 역할이 AWS MKS 키에 액세스할 수 있도록 허용했으므로 보안 암호를 순환할 수 있습니다.

  1. AWS 관리 콘솔 상단의 검색 창에서

    Copied
    를 검색하여 선택합니다.

  2. Secrets 페이지에서 dbadmin_secret 링크를 선택하여 해당 세부 정보를 확인합니다.

  3. dbadmin_secret 페이지의 Secret value 섹션에서 Retrieve secret value를 선택합니다.

password 값이 이러한 지침 왼쪽에 표시되는 DbPassword 값과 동일하고 사용자가 입력한 원래 값과 여전히 일치합니다.

  1. Rotation 섹션에서 Rotate secret immediately를 선택합니다.

  2. Rotate secret 창에서 Rotate를 선택합니다.

예상 출력: 페이지 상단에 다음과 같은 내용의 녹색 배너가 표시됩니다(표시하기 위해 위로 스크롤해야 할 수 있음). Secret scheduled for rotation.

새로 고침: 보안 암호 순환을 완료하는 데 약 1분이 소요됩니다.

  1. 1분 동안 기다린 다음, 페이지를 새로 고칩니다.

  2. Overview 탭을 선택합니다.

  3. Secret value 섹션에서 Retrieve secret value를 선택합니다.

  4. password 값이 변경되었는지 확인합니다.

참고: 암호가 변경되지 않은 경우 Rotation configuration 섹션 하단에 있는 Lambda rotation function 링크를 사용하여 Lambda 함수를 보고 로그 파일에 액세스하여 문제 해결 시 도움을 얻습니다.

보안 암호 순환에 대한 자세한 내용은 이 실습 마지막에 나오는 추가 리소스 섹션에서 AWS Secrets Manager 보안 암호 순환을 참조하십시오.

  1. 터미널 세션이 EC2 인스턴스에 연결된 상태에서 웹 브라우저 탭으로 돌아갑니다.

먼저, 원래 암호로 데이터베이스에 수동으로 연결하여 데이터베이스 내에서 변경되었는지 확인합니다.
