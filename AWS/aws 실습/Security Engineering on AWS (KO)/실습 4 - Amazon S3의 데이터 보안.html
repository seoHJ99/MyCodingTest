실습 개요

이 실습에서는 Amazon Simple Storage Service(Amazon S3)에 저장된 데이터를 보호하는 다양한 방법을 알아봅니다. 먼저 S3 버킷의 내용을 암호화하는 데 사용하는 AWS Key Management Service(AWS KMS) 키를 생성합니다. 그런 다음 버킷에 업로드된 새 객체의 암호화를 적용하는 S3 버킷 정책을 생성합니다. 그런 다음 S3 버킷의 개체에 저장된 개인 식별 정보(PII)를 식별하는 데 사용되는 Amazon Macie 작업을 생성합니다. 그런 다음 복제 규칙을 만들어 원본 버킷에서 다른 지역의 복제 버킷으로 개체를 복제하여 데이터에 다중 리전 복원력을 제공합니다. 마지막으로, Macie 작업 결과를 검토합니다.

목표

본 실습을 마치면 다음을 할 수 있습니다.

  • S3 버킷에서 사용할 AWS KMS 키 생성
  • AWS KMS 키를 사용하여 S3 버킷의 데이터 암호화
  • 업로드된 객체에 대해 암호화를 적용하는 S3 버킷 정책 생성
  • S3 버킷의 암호화된 데이터를 다른 리전의 버킷으로 복제
  • Macie를 사용하여 S3 버킷의 객체에 저장된 개인 식별 정보(PII) 식별

필수 기술 지식

시나리오

여러분은 AnyCompany의 보안 엔지니어로서 S3 버킷에 저장된 데이터를 보호하기 위한 회사의 데이터 보호 전략을 적용하는 업무를 맡았습니다. 이러한 전략에는 다음이 필요합니다.

  • S3 버킷에 저장된 데이터는 동일한 버킷 리전에서 관리하는 AWS KMS 키로 서버 측 암호화(SSE)를 사용하여 암호화됩니다.
  • S3 버킷에 업로드된 데이터는 올바른 암호화 방법과 AWS KMS 키를 사용하지 않을 경우 거부되어야 합니다.
  • 기본 리전의 S3 버킷에 저장된 데이터는 다중 리전 복원력을 제공하기 위해 보조 리전의 다른 S3 버킷에 복제되어야 합니다. 또한 이 데이터는 해당 리전의 다른 AWS KMS 키를 사용하여 보조 리전에서도 암호화해야 합니다.
  • S3 버킷에 저장된 데이터에는 개인 식별 정보(PII)가 포함되지 않아야 합니다.

첫 실습 빌드에는 다음이 포함됩니다.

  • 기본 리전의 S3 버킷(original-bucket)
  • 보조 리전의 또 다른 S3 버킷(replication-bucket)

과제 1: 기본 및 보조 리전에 AWS KMS 키 생성

이 과제에서는 2개의 AWS KMS 키를 생성합니다. 각 키는 해당 리전의 Amazon S3 버킷에 대한 서버 측 암호화(SSE)에 사용됩니다. 회사의 데이터 보호 전략의 일환으로 보안 규정 준수에서는 Amazon S3 버킷에 저장된 모든 데이터를 사용자가 관리하는 서버 측 암호화 AWS KMS 키로 암호화하도록 요구합니다.

과제 1.1: 기본 및 보조 리전 버킷 식별

AWS KMS 키를 생성하기 전에 기본 및 보조 S3 버킷을 찾아야 합니다.

  1. AWS 관리 콘솔 상단의 검색 창에서

    Copied
    를 검색하여 선택합니다.

  2. S3 Management Console에서 Buckets 아래에서 버킷이 나열된 위치를 찾습니다.

  3. 기본 리전 버킷 및 해당 리전을 기록해 둡니다. 기본 리전 S3 버킷 이름은 original-bucket-<RANDOM_NUMBER> 가 됩니다.

  4. 보조 리전 버킷 및 해당 리전을 기록해 둡니다. 보조 리전 S3 버킷 이름은 replication-bucket-<RANDOM_NUMBER> 가 됩니다.

참고: 이러한 지침 왼쪽의 목록에서 항상 기본 및 보조 버킷 및 리전 이름을 찾을 수 있습니다.

과제 2: AWS KMS 키를 사용하여 S3 버킷의 데이터 암호화

이 과제에서는 해당하는 리전 AWS KMS 키를 사용하도록 기본 및 보조 리전 버킷 둘 다에서 기본 암호화를 설정합니다. 이렇게 하면 버킷의 기본 암호화를 사용하여 업로드한 객체가 자동으로 암호화됩니다. S3 버킷에 대해 암호화를 설정한 후에 샘플 파일을 업로드하여 테스트합니다.

참고: 기본 암호화는 강제 정책이 아닙니다.업로드 시 암호화 옵션을 설정하지 않고 업로드할 객체만 암호화합니다.객체 업로드 중에 암호화를 적용하는 방법은 이후 작업에서 다루겠습니다.

과제 2.1: 해당 AWS KMS 키를 사용하여 각 버킷에서 기본 암호화 사용

  1. AWS 관리 콘솔 상단의 검색 창에서

    Copied
    를 검색하여 선택합니다.

  2. S3 Management Console의 Buckets에서 버킷이 나열된 위치를 찾고 original-bucket-<RANDOM_NUMBER> 링크를 선택하여 원본 버킷 설정에 액세스합니다.

  3. Properties 탭을 선택하고 아래로 스크롤하여 Default encryption으로 이동한 후 Edit을 선택합니다.

  4. Edit default encryption 페이지에서 다음 옵션을 구성합니다.

  • Encryption key type: Server-side encryption with AWS Key Management Service keys (SSE-KMS) 를 선택합니다.

  • AWS KMS key: Choose from your AWS KMS keys

  • Available AWS KMS keys: original-kms-key

  1. Save changes를 선택합니다.

예상 출력:

Successfully edited default encryption.

이제 원본 버킷 기본 암호화가 기본 리전 AWS KMS 키를 사용하도록 설정되었습니다.

  1. replication-bucket-<RANDOM_NUMBER> 에 대해 위의 동일한 단계를 반복하고 기본 암호화를 replication-kms-key AWS KMS 키를 사용하도록 설정합니다.

과제 2.2: 샘플 파일을 원본 버킷으로 업로드하여 암호화 설정 확인

  1. 이러한 지침 왼쪽의 목록에서 S3ObjectsLink를 복사한 후 새 브라우저 탭으로 붙여넣어 zip 파일 'files.zip’을 로컬 컴퓨터로 다운로드합니다. zip 파일에는 S3 버킷으로의 업로드를 테스트하기 위한 샘플 파일에 포함되어 있습니다.

  2. 로컬 컴퓨터에서 압축된 파일의 압축을 풉니다. 압축을 푼 폴더에는 테스트에 사용하는 샘플 파일이 포함되어 있습니다.

  3. S3 Management Console의 Buckets에서 버킷이 나열된 위치를 찾고 original-bucket-<RANDOM_NUMBER> 링크를 선택하여 원본 버킷 설정에 액세스합니다.

  4. Objects 탭에서 Upload를 선택합니다.

  5. Upload 페이지에서 Add files를 선택합니다.

  6. 팝업되는 File Upload 창에서 로컬 컴퓨터의 압축 해제된 폴더를 선택하고 업로드할 샘플 파일 중 하나를 선택합니다.

  7. Upload를 선택합니다.

예상 출력:

Upload succeeded

  1. Close를 선택합니다.

과제 5: 재해 복구 리전 버킷으로 데이터 복제

  • 이전 과제에서 구성한 버킷 정책 때문에 original-kms-key로 암호화한 원본 버킷으로만 객체를 업로드할 수 있습니다.
  • 복제 규칙은 원본 버킷의 객체를 복제 버킷으로 복제합니다.
  • 복제 규칙은 복제 전에 original-kms-key를 사용하여 원본 버킷의 객체를 암호 해독합니다.
  • 복제 규칙은 replication-kms-key를 사용하여 복제 버킷으로 복제할 때 객체를 암호화합니다.

복제 규칙이 위 과제를 모두 수행하려면 IAM 권한이 필요합니다. 이러한 권한은 해당 규칙으로 생성된 IAM 복제 서비스 역할에 추가됩니다.

힌트: 역할 이름은 S3ReplicationRole입니다.

  1. 탐색 모음의 리전 이름을 확인하고 실습 기본 리전에 있는지 확인합니다. 이러한 지침 왼쪽의 목록에서 기본 리전 값을 찾을 수 있습니다. 그렇지 않은 경우 리전 선택기를 사용하여 올바른 리전을 선택합니다.

Macie는 객체가 AWS KMS 키로 암호화된 경우에도 객체 내용을 읽을 수 있었습니다. Macie 서비스 역할에 객체를 암호 해독할 권한을 부여했기 때문입니다.
