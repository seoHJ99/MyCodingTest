개요

개발 환경을 설정하는 방법을 익혔으니 이제 Amazon S3를 스토리지 서비스 및 웹 호스팅 서비스로 사용하는 애플리케이션의 일부를 개발하겠습니다. 이 애플리케이션은 Amazon S3를 사용하여 다양한 목적으로 여러 형식의 파일을 저장합니다. 이런 형식에는 csv와 json은 물론 이후 실습의 mp3 파일도 포함됩니다. Amazon S3를 사용하여 정적 웹 사이트도 호스트합니다. 주어진 단서를 사용하여 TODO 자리 표시자에서 코드의 누락된 부분을 채웁니다. 오류가 없는 코드를 채운 다음 프로그램을 실행하고 결과를 검토합니다.

이 실습을 완료하면 다음을 할 수 있게 됩니다.

  • AWS SDK 및 AWS CLI를 사용하여 프로그래밍 방식으로 Amazon S3와 상호 작용합니다.
  • waiter를 사용하여 버킷을 생성하고 서비스 예외 코드를 확인합니다.
  • 메타데이터가 첨부된 Amazon S3 객체를 업로드하는 데 필요한 요청을 빌드합니다.
  • 버킷에서 객체를 다운로드하는 요청을 빌드하고, 데이터를 처리하고, 객체를 버킷에 다시 업로드합니다.
  • AWS CLI를 사용하여 웹 사이트를 호스트하고 소스 파일을 동기화하도록 버킷을 구성합니다.

과제 1: 새 Amazon S3 버킷 생성

이 과제는 실습 환경에서 실행 중인 Visual Studio를 사용하여 수행합니다. 실습 리전에서 고유한 이름으로 notes-bucket을 생성하는 코드를 개발합니다. 시간을 절약하기 위해 각 스크립트의 기본 코딩 구성 요소가 제공되었으며, 누락된 코드는 추가해야 합니다.

.NET용 AWS SDK 설명서를 사용하여 제공된 프로젝트에서 TODO로 표시된 코드를 완성합니다. 상세 지침에는 여러 보기가 있습니다.

이 실습의 콘솔 애플리케이션 프로젝트를 지원하는 클래스는 일관성을 제공하기 위해 같은 순서로 배열되어 있으므로 보다 읽기 쉽습니다. Amazon S3용 SDK 패키지 AWSSDK.S3 등 필요한 NuGet 패키지 종속성은 이미 프로젝트 파일에 추가되었습니다. 각 클래스마다 과제의 단계를 실행하기 위해 호출되는 Run() 메서드 및 모든 지원 메서드와 함께 네임스페이스 선언이 포함됩니다. 메서드가 독립적이고 향후 테스트 및 문제 해결이 쉽도록 각 메서드에는 필요한 모든 값이 전달됩니다.

.NET Core는 태스크 기반 비동기식 프로그래밍(TAP) 모델을 사용합니다. SDK에서 AWS 서비스 작업에 대한 메서드 호출은 HTTPS 엔드포인트에 대한 호출로 구현되므로 비동기로 표시된 메서드 내에서 대기됩니다. 이러한 호출의 결과를 반환하는 메서드는 결과를 태스크 객체로 적절히 래핑합니다.

모든 로직이 Run() 메서드에서 처리되므로 메서드의 코드는 try/catch 블록으로 래핑됩니다. 전체 클래스의 오류는 이 수준에서 처리됩니다.

아이콘 설명

이 실습에서는 다양한 유형의 지침 및 참고 사항에 대한 주의를 환기하기 위해 다양한 아이콘이 사용됩니다. 모든 아이콘이 사용되지는 않지만 다음 목록에서는 각 아이콘의 용도를 설명합니다.

  • 명령: 실행해야 하는 명령입니다.
  • 예상 출력: 명령 또는 편집된 파일의 출력을 확인하는 데 사용할 수 있는 샘플 출력입니다.
  • 참고: 참고 사항, 팁 또는 중요한 가이드입니다.
  • 추가 정보: 자세한 정보를 찾을 수 있는 위치입니다.
  • 주의: 특별한 관심이 필요한 중요한 정보입니다(놓쳤다고 해도 장치 또는 데이터에 문제가 발생할 정도로 중요하지는 않지만 특정 단계를 반복해야 할 수 있음).
  • 경고: 되돌릴 수 없고, 명령 또는 프로세스의 실패에 영향을 줄 수 있는 작업에 대해 주의하도록 합니다(구성을 설정한 후 변경할 수 없는 구성에 대한 경고가 포함됨).
