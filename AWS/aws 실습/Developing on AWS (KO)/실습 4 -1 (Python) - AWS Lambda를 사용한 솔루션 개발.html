개요

Amazon DynamoDB 테이블에 노트를 저장했으니 이제 이와 상호 작용할 애플리케이션 로직을 생성할 수 있습니다. 이전 실습에서는 AWS SDK를 사용하여 AWS 서비스와 상호 작용했습니다. 이제는 개발하고 있었던 코드를 실행할 수 있는 위치를 살펴봐야 합니다.

AWS의 Amazon Elastic Compute Cloud(Amazon EC2)를 사용하여 운영 시스템 및 인스턴스 크기에 대한 완전한 제어가 필요한 경우 코드를 실행할 수 있습니다. 이 정도의 제어가 필요하지 않은 경우 언어를 지원하는 사전 빌드된 런타임 환경에서 코드를 실행할 수 있습니다. 컨테이너의 오케스트레이션을 제어해야 하는 경우 Amazon Elastic Container Service(Amazon ECS) 또는 Amazon Elastic Kubernetes Service(Amazon EKS)를 사용할 수 있습니다. 하지만 최소 구성으로 코드를 실행하고자 하는 경우 AWS Lambda를 사용하고자 합니다.

이 실습에서는 AWS Lambda 함수를 사용하여 코드를 배포함으로써 애플리케이션에 대한 백엔드 로직을 생성합니다.

과제 2: Lambda 함수 코드에 처리 로직 추가

AWS Cloud9 환경에서 가져온 모든 종속성, 생성된 클라이언트와 리소스 및 생성된 사용자 파라미터가 포함된 기본 함수가 제공되었습니다. 이 함수는 Amazon S3 버킷에 호스트된 MP3 파일에 대한 액세스를 부여하는 서명된 URL을 반환합니다. MP3 파일은 데이터베이스에 저장된 노트의 오디오 버전입니다.

이 함수는 3개의 주요 단계를 거칩니다.

  • Amazon DynamoDB에서 노트의 텍스트를 검색합니다.
  • Amazon Polly를 사용하여 텍스트를 음성으로 변환하고 오디오 스트림을 로컬 파일로 저장합니다.
  • Amazon S3에 파일을 업로드하고 함수로부터 반환할 미리 서명된 URL을 생성합니다.

과제 3: AWS Lambda 함수 게시

AWS Lambda 함수용 코드는 스크립트 또는 컴파일된 프로그램과 그 종속성으로 구성되어 있습니다. 배포 패키지를 사용하여 함수 코드를 Lambda로 배포합니다. Lambda는 컨테이너 이미지와 .zip 파일 아카이브라는 두 가지 유형의 배포 패키지를 지원합니다.

.zip 파일 아카이브에는 애플리케이션 코드와 그 종속성이 포함되어 있습니다. Lambda 콘솔이나 툴키트를 사용하여 함수를 작성하는 경우 Lambda는 자동으로 코드에 대한 .zip 파일 아카이브를 생성합니다.

Lambda API, 명령줄 도구 또는 AWS SDK를 사용하여 함수를 생성한 경우 배포 패키지를 생성해야 합니다. 함수에서 컴파일된 언어를 사용하는 경우, 또는 함수에 종속성을 추가하기 위해 배포 패키지를 생성해야 합니다. 함수의 코드를 배포하기 위해 Amazon Simple Storage Service(Amazon S3) 또는 로컬 컴퓨터에서 배포 패키지를 업로드합니다.

참고: 이 실습에서 사용되는 모든 종속성은 AWS Lambda Python 3.11 런타임에 이미 포함되어 있으며 배포 패키지에 수동으로 추가할 필요가 없습니다.

개략적 지침
