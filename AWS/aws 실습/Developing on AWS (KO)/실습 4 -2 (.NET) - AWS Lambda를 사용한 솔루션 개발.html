개요

Amazon DynamoDB 테이블에 노트를 저장했으니 이제 파일과 상호 작용할 애플리케이션 로직을 생성할 수 있습니다. 이전 실습에서는 AWS SDK를 사용하여 AWS 서비스와 상호 작용했습니다. 이제 개발해 왔던 코드를 실행합니다.

AWS의 Amazon Elastic Compute Cloud(Amazon EC2)를 사용하여 운영 시스템 및 인스턴스 크기에 대한 완전한 제어가 필요한 경우 코드를 실행할 수 있습니다. 이 정도의 제어가 필요하지 않은 경우 언어를 지원하는 사전 빌드된 런타임 환경에서 코드를 실행할 수 있습니다. 컨테이너의 오케스트레이션을 제어해야 하는 경우 Amazon Elastic Container Service(Amazon ECS) 또는 Amazon Elastic Kubernetes Service(Amazon EKS)를 사용할 수 있습니다. 하지만 최소 구성으로 코드를 실행하고자 하는 경우 AWS Lambda를 사용합니다.

이 실습에서는 AWS Lambda 함수를 사용하여 코드를 배포함으로써 애플리케이션에 대한 백엔드 로직을 생성합니다.

목표

이 실습을 완료하면 다음을 할 수 있게 됩니다.

  • AWS Lambda 함수를 생성하고 AWS SDK 및 AWS CLI를 사용하여 프로그래밍 방식으로 상호 작용합니다.
  • Lambda 함수를 구성하여 환경 변수를 사용하고 다른 서비스와 통합합니다.
  • AWS SDK를 사용하여 Amazon S3 미리 서명된 URL을 생성하고 버킷 객체에 대한 액세스를 확인합니다.
  • IDE를 통해 .zip 파일 아카이브를 사용하여 Lambda 함수를 배포하고 필요한 경우 테스트합니다.
  • AWS Console 및 AWS CLI를 사용하여 AWS Lambda 함수를 호출합니다.

과제 2: AWS Lambda 함수 코드에 처리 로직 추가

제공된 환경에는 dictate-function의 스켈레톤 버전 코드를 포함하여 이 실습의 Lambda 함수에 대한 모든 프로젝트 및 코드가 포함된 솔루션 파일이 있습니다. 이 스켈레톤 함수에는 모든 필요한 NuGet 패키지 종속성 및 네임스페이스에 대한 참조는 물론 함수에 제공된 사용자 파라미터가 포함되어 있습니다. 이 함수는 Amazon S3 버킷에 호스트된 MP3 파일에 대한 액세스를 부여하는 서명된 URL을 반환합니다. MP3 파일은 데이터베이스에 저장된 노트의 오디오 버전입니다.

이 함수는 아래에 있는 3개의 주요 단계를 완료합니다.

  • Amazon DynamoDB에서 노트의 텍스트를 검색합니다.
  • Amazon Polly를 사용하여 텍스트를 음성으로 변환하고 오디오 스트림을 로컬 파일로 저장합니다.
  • Amazon S3에 파일을 업로드하고 함수로부터 반환할 미리 서명된 URL을 생성합니다.

아이콘 설명

이 실습에서는 다양한 유형의 지침 및 참고 사항에 대한 주의를 환기하기 위해 다양한 아이콘이 사용됩니다. 모든 아이콘이 사용되지는 않지만 다음 목록에서는 각 아이콘의 용도를 설명합니다.

  • 명령: 실행해야 하는 명령입니다.
  • 예상 출력: 명령 또는 편집된 파일의 출력을 확인하는 데 사용할 수 있는 샘플 출력입니다.
  • 참고: 참고 사항, 팁 또는 중요한 가이드입니다.
  • 추가 정보: 자세한 정보를 찾을 수 있는 위치입니다.
  • 주의: 특별한 관심이 필요한 중요한 정보입니다(놓쳤다고 해도 장치 또는 데이터에 문제가 발생할 정도로 중요하지는 않지만 특정 단계를 반복해야 할 수 있음).
  • 경고: 되돌릴 수 없고, 명령 또는 프로세스의 실패에 영향을 줄 수 있는 작업에 대해 주의하도록 합니다(구성을 설정한 후 변경할 수 없는 구성에 대한 경고가 포함됨).
  • 고려 사항: 자신의 환경에 개념을 적용하는 방법을 고려하거나 당면한 주제에 대한 대화를 시작하기 위해 일시 중지할 시점을 제안합니다.
  • 복사/붙여넣기: 복사 및 붙여넣기가 필요한 미리 생성된 스크립트 또는 파일의 내용을 표시하는 코드 블록입니다. 코드 블록의 특정 부분만 복사해야 하는 경우 코드에 번호가 매겨진 TODO 주석이 있습니다.
  • 이해도 점검: 지식을 확인하고 학습한 내용을 테스트하는 기회입니다.
  • 보안: 보안 모범 사례를 적용할 기회입니다.
  • 새로 고침: 웹 브라우저 페이지 또는 목록을 새로 고쳐 새 정보를 표시해야 하는 경우입니다.
  • 복사 명령: (특정 변수를 편집하기 위해) 명령, 스크립트 또는 기타 텍스트를 텍스트 편집기로 복사하는 것이 명령줄 또는 터미널에서 직접 편집하는 것보다 쉬울 수 있는 경우입니다.
  • 힌트: 질문 또는 도전 과제에 대한 힌트입니다.
  • 해답: 질문 또는 도전 과제의 해답입니다.
  • 그룹 활동: 과제를 완료하기 위해 다른 수강생과 함께 작업해야 하는 경우입니다.
    • 코드 블록의 특정 부분만 복사해야 하는 경우 코드에 번호가 매겨진 TODO 주석이 있습니다.
