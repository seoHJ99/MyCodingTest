개요

Amazon DynamoDB 테이블에 노트를 저장했으니 이제 파일과 상호 작용할 애플리케이션 로직을 생성할 수 있습니다. 이전 실습에서는 AWS SDK를 사용하여 AWS 서비스와 상호 작용했습니다. 이제 개발해 왔던 코드를 실행합니다.

AWS의 Amazon Elastic Compute Cloud(Amazon EC2)를 사용하여 운영 시스템 및 인스턴스 크기에 대한 완전한 제어가 필요한 경우 코드를 실행할 수 있습니다. 이 정도의 제어가 필요하지 않은 경우 언어를 지원하는 사전 빌드된 런타임 환경에서 코드를 실행할 수 있습니다. 컨테이너의 오케스트레이션을 제어해야 하는 경우 Amazon Elastic Container Service(Amazon ECS) 또는 Amazon Elastic Kubernetes Service(Amazon EKS)를 사용할 수 있습니다. 하지만 최소 구성으로 코드를 실행하고자 하는 경우 AWS Lambda를 사용합니다.

이 실습에서는 AWS Lambda 함수를 사용하여 코드를 배포함으로써 애플리케이션에 대한 백엔드 로직을 생성합니다.

목표

이 실습을 완료하면 다음을 할 수 있게 됩니다.

  • AWS Lambda 함수를 생성하고 AWS SDK 및 AWS CLI를 사용하여 프로그래밍 방식으로 상호 작용합니다.
  • Lambda 함수를 구성하여 환경 변수를 사용하고 다른 서비스와 통합합니다.
  • AWS SDK를 사용하여 Amazon S3 미리 서명된 URL을 생성하고 버킷 객체에 대한 액세스를 확인합니다.
  • IDE를 통해 .zip 파일 아카이브를 사용하여 Lambda 함수를 배포하고 필요한 경우 테스트합니다.
  • AWS Console 및 AWS CLI를 사용하여 AWS Lambda 함수를 호출합니다.

과제 1: AWS Lambda 함수 생성

첫 번째 함수에서는 저장한 노트를 필요 시 음성파일을 생성합니다. 이 함수는 Amazon DynamoDB, Amazon Polly 및 Amazon S3를 사용합니다.

사용 가능한 옵션의 표현을 볼 수 있는 AWS Console을 사용하여 실습을 시작합니다. 하지만 이 실습의 대부분은 AWS CLI와 코드에서 수행됩니다.

불완전한 소스 코드가 제공됩니다. 제공된 단서를 활용하여 TODO 자리 표시자의 코드에서 누락된 부분을 완성합니다. 구문 오류가 없는 코드를 채운 다음 코드를 실행하여 결과를 검토합니다.

개략적 지침

  • AWS Console을 사용하여
    Copied
    을 생성합니다.
  • 런타임을 Java 11 (Corretto) 로 설정합니다.
  • lambdaPollyRole을 사용하여 Lambda 함수에 권한을 부여합니다.

상세 지침

  1. AWS Management Console에서 AWS 검색 창을 사용하여

    Copied
    를 검색한 다음 결과 목록에서 서비스를 선택합니다.

  2. Create function을 선택합니다.

  3. Author from scratch 를 선택합니다.

  4. Function name에

    Copied
    을 입력합니다.

  5. Runtime에는 Java 11 (Corretto) 을 선택합니다.

  6. Change default execution role을 확장하고 Use an existing role을 선택합니다.

  7. Existing role에서 lambdaPollyRole을 선택합니다.

  8. Create function을 선택합니다.

AWS Lambda 콘솔을 사용하여 Lambda 함수를 생성, 구성, 모니터링 및 호출할 수 있습니다. 함수 코드가 컴파일을 필요로 하지 않는 언어(Node.js, Python 또는 Ruby)로 작성되고 3MB 미만인 경우 콘솔에서 함수 코드를 보고 수정할 수 있습니다. lambdaPollyRole은 함수에 DynamoDB, Amazon S3, Amazon Polly 및 Logs 서비스에 대한 액세스 권한을 제공합니다.

축하합니다! 이름이 dictate-function인 Lambda 함수를 생성했습니다.

과제 2: AWS Lambda 함수 코드에 처리 로직 추가

Lambda 함수 핸들러는 이벤트를 처리하는 함수 코드의 메서드입니다. 함수가 호출되면 Lambda에서 핸들러 메서드를 실행합니다. 핸들러가 존재하거나 응답을 반환할 때, 또 다른 이벤트를 처리하는 데 사용할 수 있게 됩니다.

Handler라는 클래스는 handleRequest라는 핸들러 메서드를 정의합니다. 핸들러 메서드는 이벤트 및 컨텍스트 객체를 입력으로 가져오고 문자열을 반환합니다.

InetelliJ를 사용하는 개발 환경에서는 가져온 모든 종속성, 생성된 클라이언트와 리소스 및 생성된 사용자 파라미터가 포함된 기본 함수가 제공되었습니다. 이 함수는 Amazon S3 버킷에 호스트된 MP3 파일에 대한 액세스를 부여하는 서명된 URL을 반환합니다. MP3 파일은 데이터베이스에 저장된 노트의 오디오 버전입니다.

이 함수는 아래에 있는 3개의 주요 단계를 완료합니다.

  • Amazon DynamoDB에서 노트의 텍스트를 검색합니다.
  • Amazon Polly를 사용하여 텍스트를 음성으로 변환하고 오디오 스트림을 로컬 파일로 저장합니다.
  • Amazon S3에 파일을 업로드하고 함수로부터 반환할 미리 서명된 URL을 생성합니다.

개략적 지침

  • 값을 할당하여 2개의 환경 변수를 사용합니다.

    • MP3_BUCKET_NAME(이름에 apibucket을 포함하여 실습 환경에서 버킷 이름 사용)
    • TABLE_NAME(Notes)

  • 개발 환경에서 각 함수에 전달된 파라미터를 사용하여 java[dictate] > src > main > java > dictate 폴더의

    Copied
    를 완료합니다.

    • Amazon DynamoDB로부터 텍스트를 반환하는 TODO 1에 코드를 추가합니다.
    • Amazon Polly를 사용하여 노트의 오디오 버전이 있는
      Copied
      폴더에서 로컬 MP3 파일을 생성하고 파일 위치로 반환하는 TODO 2에 코드를 추가합니다.
    • Amazon S3에 MP3 파일을 업로드하고, 로컬 파일을 제거하고, 객체에 미리 서명된 URL을 반환하는 TODO 3에 코드를 추가합니다.

과제 2.1: 코드를 추가하여 Amazon DynamoDB로부터 노트 텍스트 검색

  1. RDP 세션으로 돌아가 개발 환경에 액세스합니다.

  2. IntelliJ Project 메뉴에서 dictateFunction[dictate] > src > main > java > dictate를 확장하고 Handler를 엽니다.

이 클래스는 가져온 모든 필요한 종속성으로 시작됩니다. 다음 섹션은 성능 개선을 위해 AWS 모범 사례를 따르는 핸들러 함수 외부의 SDK 클라이언트 및 리소스를 초기화합니다. 다음은 Lambda 함수가 호출될 때 호출되는 핸들러 함수입니다.

함수는 소스에서 JSON 페이로드를 읽고, DynamoDB를 호출하여 userId 및 noteId에서 텍스트를 가져오고, Polly synthesize_speech API를 호출하여 텍스트를 음성으로 변환하고, 그 결과로 나온 오디오를 /tmp 폴더에 MP3 파일로 저장하고, 해당 MP3 파일을 Amazon S3에 업로드하고, MP3 파일에 대해 미리 서명된 URL을 생성한 다음 URL을 응답 객체로 반환합니다.

핸들러 함수는 2개의 환경 변수를 참조합니다. table name을 참조하여 사용자의 노트를 읽고 bucket name을 참조하여 오디오 변환된 노트의 MP3 파일을 업로드합니다.

우선 이러한 환경 변수를 사용하도록 Lambda 구성을 변경합니다.

  1. 복사/붙여넣기: 버킷이 이미 생성되어 있습니다. 다음 단계에서 사용하게 되므로 지침 왼쪽에 MP3BucketName의 값을 복사합니다.

  2. 명령: IntelliJ 콘솔에서 Terminal 세션(콘솔 왼쪽 하단에 있음)을 열고 다음 명령을 사용하여 환경 변수 TABLE_NAME 및 MP3_BUCKET_NAME을 할당합니다.

참고: <MP3BucketName> 자리 표시자를 지침 왼쪽에 있는 값으로 업데이트해야 합니다.

Copied 
set env-parms=TABLE_NAME=Notes, MP3_BUCKET_NAME=<MP3BucketName>

과제 3: AWS Lambda 함수 게시

AWS Lambda 함수용 코드는 스크립트 또는 컴파일된 프로그램과 그 종속성으로 구성되어 있습니다. 배포 패키지를 사용하여 함수 코드를 Lambda로 배포합니다. Lambda는 컨테이너 이미지와 .zip 파일 아카이브라는 두 가지 유형의 배포 패키지를 지원합니다.

.zip 파일 아카이브에는 애플리케이션 코드와 그 종속성이 포함되어 있습니다. Lambda 콘솔이나 툴키트를 사용하여 함수를 작성하는 경우 Lambda는 자동으로 코드에 대한 .zip 파일 아카이브를 생성합니다.

Lambda API, 명령줄 도구 또는 AWS SDK를 사용하여 함수를 생성한 경우 배포 패키지를 생성해야 합니다. 함수에서 컴파일된 언어를 사용하는 경우, 또는 함수에 종속성을 추가하기 위해 배포 패키지를 생성해야 합니다. 함수의 코드를 배포하기 위해 Amazon Simple Storage Service(Amazon S3) 또는 로컬 컴퓨터에서 배포 패키지를 업로드합니다.
