클러스터 크기 조정 제어
CloudWatch 의 CapacityProviderReservation 지표를 기반으로 크기를 조정
목표 인스턴스 / 현재 실행중인 인스턴스 * 100
만약 100 보다 크면 늘림. 작으면 줄임.

tini 또는 dumb-init 과 같은 프로세스 관리자를 이용해야 정상적으로 컨테이너 프로세스 종료 가능

cpu 사용량 기반 또는 작업자 기반으로 조정
cpu 바인딩
    - cpu 사용량을 기준으로 늘리거나 줄임
    - 장애 대처가 중심인 경우 40% 사용량 가용영역 두개 권장
작업자 기반
    - 사용자 요청에 따라 늘리거나 줄임
    - SQS 대기열 메시지 처리 방식
    - 클러스터 용량보다는 서비스 용량을 오버프로비저닝 하는걸 권장
    - 메시지 깊이를 기준삼아 늘리거나 줄임
