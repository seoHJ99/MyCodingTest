모듈4. 컴퓨팅

----------------------
컴퓨팅 서비스
ec2, 람다, 컨테이너 서비스 등이 존재함

-------------------------
ec2 인스턴스
전세계 aws 리전에 물리적 서버에 호스트됨
api 를 호출하여 인스턴스 생성 가능
시작 고려사항
1. 태그 : 키-값으로 관리되는 태그. 한번에 관리하기 편함
2. AMI - 연결할 블록을 지정하는 블록 디바이스 매핑 가능.
3. 유형. c6g.xlarge 이때 c는 인스턴스 패밀리, 6은 세대, g는 추가 속성 .뒤에는 크기
    - 최신 세대일수록 같은 가격이어도 더 선응이 좋음
4. 키페어 - ssh 로 연결될때 사용가능. 선택사항
5. 테넌시 : 인스턴스가 실행될 하드웨어를 선택 가능.
6. 배치그룹 
    1. 클러스터 : ec2 인스턴스를 서로 인접시켜 적은 지연시간
    2. 분산형 : 내결함성
    3. 파티션 : 논리그룹으로 분할. 상관 관계가 있는 하드웨어 장애를 방지
7. 사용자 데이터 : 일반적으로 자동구성 태스크를 수행하는 사용
    - 스크립트를 사용
8. 인스턴스 메타데이터 : 실행중인 ec2 인스턴스 정보

------------------
ec2 인스턴스용 스토리지
1. ebs
    - 하나의 인스턴스에 연결됨
    - 필요에 따라 인스턴스간 ebs 이동이 가능
    - 크게 ssd 와 hdd 로 나뉨
    - ssd 는 범용과 추가적인 프로비저닝이된 유형이 존재함
    - hdd 는 처리량 최적화 hdd 와 콜드 hdd로 나뉨
2. 인스턴스 스토어
    - 호스트에 연결되어 있어 인스턴스를 중지, 종료하면 사라짐

---------------------
EC2 요금 옵션

1. 온디맨드
   - 기본. 약정없이 사용량만큼 지불
2. saving plans
   - 1년 또는 3년 약정.
   - compute savings plans : 컴퓨팅, 테넌시, 리전 등 변경 가능
   - instance savings plans : 컴퓨팅 옵션만 변경 가능
3. 스팟 인스턴스
   - 온디맨드 비용에서 최대 90프로 할인
   - 종료 2분전 알람. 작동 중단에 대한 계획이 필요할수도 있음
   - 내결함성이 있거나 스테이트리스

--------------------
람다

서버리스 컴퓨팅 서비스
런타임 환경 선택후, 코드만 전달하면 됨
람다 함수는 최대 15분간 실행 가능.
확장 고가용성 알아서 관리
직접 호출하거나 다른 이벤트가 트리거가 되어 호출 가능.
