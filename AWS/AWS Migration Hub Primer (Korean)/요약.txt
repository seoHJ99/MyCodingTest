Migration Hub : 마이그레이션 진행 상황을 볼수 있도록 제공되는 aws 콘솔
    - 홈 리전을 선택해야 하며, 다른 리전에서 이뤄지는 마이그레이션도 확인 가능
    - 홈 리전은 변경 불가능
    - 홈 리전 외부의 쓰기 작업(알림, 생성, 연결, 가져오기 등) api 는 거절
    - 홈 리전 외부의 읽기 작업(나열, 설명, 중지, 삭제) api 는 허용
    - Migration Hub api 는 홈 리전 내부에서만 호출 가능

Discover(검색) 단계

기존 데이터 센터에서 aws 클라우드로 어플레케이션의 마이그레이션 계획 수행 가능.
여러 데이터 원본에서 데이터를 수집하고 중심도구에서 분석하여 서버를 에플리케이션으로 그룹화하여 계획 수립
   -검색 : 기존 데이터 센터에서 에플을 검색하는 기능
   -계획 : 어플을 마이그레이션 포트폴리오로 그룹화
   -마이그레이션 : 마이그레이션 진행 상황 추적

AWS Application Discovery Agent
검색과 마이그레이션을 대상으로 하는 설치 프로그램. 홈 리전에 연결되며 Application Discovery Service에 등록됨
전송 계층 보안 암호화 사용.
오프라인에서도 데이터 캡쳐 가능.

구현 순서
    1. 액세스 키로 iam 사용 생성.
    2. 마이그레이션 홈 리전 생성
    3. Discovery Agent 설치
    4. Migration Hub 를 통해 수집된 데이터 확인

15초마다 데이터 수집. 15분마다 hub 와 동기화

Application Discovery Service Agentless Collector 사용
에이전트 설치 없이 다수의 가상 머신의 구성 정보를 수집한다.

구현 순서
    1. Agentless Collector 어플라이언스 OVA 파일을 다운로드
    2. vCenter에서 Agentless Collector 어플라이언스를 배포
    3. 웹 인터페이스를 통해 Agentless Collector를 구성
    4. 데이터 수집을 시작

수동으로 서버 데이터 가져오기
csv 파일로 서버 데이터를 가져올수 있으며, 쉼표로 구분하여 파일 여러개를 올릴수도 있따.
옳바른 형식의 csv 파일 템플릿은 Migration hub 에서 다운 가능
s3 에 업로드하고, 이를 Migration hub 에서 import 해야함.

평가 단계

클라우드 준비 상태를 평가하고 IT 포트폴리오의 콘텐츠와 비용을 판단

Amazon EC2 인스턴스 권장 사항 
AWS에서 기존 서버와 애플리케이션 워크로드를 실행하는 비용을 추산하는 기능을 제공
CSV 파일로 권장 사항을 생성
가장 최근의 완전한 데이터를 사용해 인스턴스 권장 사항을 제공

마이그레이션 리소스 태깅
마이그레이션된 리소스에 태그를 할당하는 것
태그를 사용해서 리소를 쉽게 분류하고 관리할수 있다.
비용 할당 태그 : 비용 추적이 가능. aws 생성 또는 billing console 에서 활성화 가능


Migration Hub Strategy Recommendations
애플리케이션에 대한 권장 사항과 여러 트랜스포메이션 경로 비교를 제공하여 마이그레이션 및 애플리케이션 현대화 이니셔티브를 계획할 수 있도록 지원
모든 Windows 서버에 작동할 수 있는 일련의 자격 증명 및 모든 Linux 서버에 작동할 수 있는 일련의 자격 증명을 사용

Migration Hub Strategy Recommendations 애플리케이션 데이터 수집기는 온프레미스 VMware 환경에 설치하는 가상 어플라이언스
배포방법
1. VMware vCenter Server에 가상 머신(VM)으로 배포하는 방법
2.Strategy Recommendations 수집기를 Amazon Machine Image(AMI)로 사용하는 방법

외부에서 csv 파일로 Strategy Recommendations에 데이터를 가져올수도 있음

기본 지정 설정에서 우선 순위 선택 가능
라이선스 비용 절감
마이그레이션 속도
클라우드 네이티브 기술을 비롯한 인프라 현대화
관리형 서비스를 통한 운영 오버헤드 절감
