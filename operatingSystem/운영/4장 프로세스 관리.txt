프로세스 관리
프로세스 생성
부모 프로세스가 자식 프로세스 생성
프로세스의 트리(계층구조) 형성
프로세스는 자우너을 필요로 함
  운영체제로부터 받는다.
  부모와 공유한다.
자원의 공유
  모무와 자식이 모든 자원을 공유하는 모델
  일부를 공유하는 모델
  전혀 공유하지 않는 모델
수행
  부모와 자식이 공존하며 수행되는 모델
  자식이 종료될때까지 부모가 기다리는 모델
주소공간
  자식은 부모의 공간을 복사함(os 데이터도 복사됨)
  자식은 그 공간에 새로운 프로그램을 올림
유닉스의 예
  fork() 시스템 콜이 새로운 프로세스를 생성
  부모를 그대로 복사(os data except pid + binary)
  주소 공간 할당
fork 다음에 이어지는 exec() 시스템 콜을 통해 새로운 프로그램을 메모리에 올림
사용자 프로세스가 직접 생성하는 것이 아닌 운영체제를 통해서만 생성 가능.

프로세스가 마지막 명령을 수행한 수 운영체제에게 이를 알려줌(exit) 자발적인 종료
 자식이 부모에게 output data를 보냄(via wait). 항상 부모보다 자식이 먼저 종료
 프로세스의 각종 자원들이 운영체제에게 반납됨
부모 프로세스가 자식의 수행을 종료시킴(abort) 강제적인 종료
  자식이 자원의 한계치를 넘어섬
  자식에게 할당된 태스크가 더이상 필요하지 않음. 즉, 자식에게 시킬 일이 없음.
  부모가 종료(exit)하는 경우
    운영체제는 부모 프로세스가 종료하는 경우 자식이 더이상 수행되도록 두지 않는다.
    단계적 종료
